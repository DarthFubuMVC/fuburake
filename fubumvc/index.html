FubuMVC Applications

To enable Katana hosting for either FubuMVC or FubuTransportation applications, simple use the FubuRake::MvcApp helper like this snippet:

@solution = FubuRake::Solution.new do |sln|

end

FubuRake::BottleServices.new({
  # Use this option to use a different prefix for the 
  # generated tasks
  :prefix => "service", 

  # This option needs to point at the directory of the build
  # products of the service project
  :dir => "src/DiagnosticsHarness/bin/#{@solution.compilemode}", 
  :name => 'ft-harness', 
  :instance => 'something',
  :user => 'user name',
  :password => 'password',
  :autostart => true,
  :manual => true,
  :disabled => true,
  :delayed => true,
  :local_service => true,
  :network_service => true,
  :interactive => true,
  :description => 'something'
  :manual => true
})

Which will give you these rake tasks using music as the task prefix from the snippet above:

rake harness:alias # Add the alias for src/DiagnosticsHarness rake harness:chrome # run the application with Katana hosting and 'watc... rake harness:firefox # run the application with Katana hosting and 'watc... rake harness:restart # touch the web.config file to force ASP.Net hostin... rake harness:run # run the application with Katana hosting

Task Description
music:alias Creates a Bottles alias for the application directory
music:chrome Runs the application in Katana and opens the default page in Chrome in _watched_ mode
music:firefox Runs the application in Katana and opens the default page in Firefox in _watched_ mode
service:start Attempts to start the named service if it is installed
music:restart Touches the web.config file of the application to force the application to recycle if running under IIS/IISExpress
music:run Runs the application in Katana and opens the default url in the default browser
